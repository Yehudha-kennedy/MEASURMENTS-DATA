# MEASUREMENTS-DATA

## Description
Machine Learning pipeline for S11 measurement data classification. It uses a Cartesian (Real/Imaginary) representation instead of Polar (Magnitude/Phase) to avoid artifacts generated by phase-wrapping.

The project evaluates models across 3 levels of complexity, seeking an optimal balance between performance and edge-device constraints:
1. SVM / Logistic Regression
2. Gradient Boosted Trees
3. 1D CNN

## Project Structure

```text
MEASUREMENTS-DATA/
├── src/          # Source code
├── mdls/         # Trained models and model artifacts
├── figures/      # Generated plots and visualizations
├── outputs/      # General outputs and logs
├── .gitignore    # Files ignored by git
└── README.md     # Project documentation
```

## Environment Variables

This project requires certain environment variables to run, which should be defined in a `.env` file at the root directory (this file is not included in version control).

Configuration example:
```env
DATA_PATH=./data/raw
MODEL_OUTPUT_DIR=./mdls
FIGURES_OUTPUT_DIR=./figures
OUTPUTS_DIR=./outputs
RANDOM_SEED=42
```
